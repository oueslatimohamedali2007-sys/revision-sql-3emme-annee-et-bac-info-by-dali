Chapitre 3

Contraintes d'intégrité

Règles strictes de protection de la cohérence et de la validité des données.

PRIMARY KEY — Clé Primaire

Identifie de façon unique chaque ligne. Une seule par table, non NULL et unique.

-- Lors de la création
CREATE TABLE eleves (
  id INT PRIMARY KEY,
  nom VARCHAR(50)
);

-- Ou après la création
ALTER TABLE eleves ADD PRIMARY KEY (id);

-- Avec auto-incrémentation
CREATE TABLE eleves (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nom VARCHAR(50)
);

NOT NULL — Valeur Obligatoire

Garantit que la colonne doit toujours avoir une valeur (pas vide).

CREATE TABLE eleves (
  id INT PRIMARY KEY,
  nom VARCHAR(50) NOT NULL,
  prenom VARCHAR(50),
  email VARCHAR(100) NOT NULL
);

-- ❌ Erreur: on ne peut pas insérer NULL
INSERT INTO eleves (id, nom, email) VALUES (1, NULL, 'test@email.com');

UNIQUE — Valeur Unique

Chaque ligne doit avoir une valeur différente (pas de doublon).

CREATE TABLE eleves (
  id INT PRIMARY KEY,
  nom VARCHAR(50),
  email VARCHAR(100) UNIQUE,
  numero_inscription INT UNIQUE
);

-- ❌ Erreur: doublon d'email
INSERT INTO eleves (id, email) VALUES (1, 'ali@mail.com');
INSERT INTO eleves (id, email) VALUES (2, 'ali@mail.com'); -- Doublon!

CHECK — Vérification de Valeur

Vérifie que la valeur respecte une condition logique.

CREATE TABLE eleves (
  id INT PRIMARY KEY,
  nom VARCHAR(50),
  note INT CHECK (note >= 0 AND note <= 20),
  age INT CHECK (age >= 3 AND age <= 100)
);

-- ❌ Erreur: note hors limites
INSERT INTO eleves VALUES (1, 'Ali', 25, 15); -- 25 > 20!

DEFAULT — Valeur Par Défaut

Attribue automatiquement une valeur si aucune n'est fournie.

CREATE TABLE eleves (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nom VARCHAR(50),
  statut VARCHAR(20) DEFAULT 'Inscrit',
  date_inscription DATE DEFAULT CURDATE(),
  actif BOOLEAN DEFAULT TRUE
);

-- La date sera remplie automatiquement
INSERT INTO eleves (nom) VALUES ('Ali');

FOREIGN KEY — Clé Étrangère (Relation)

Lie une colonne d'une table à la clé primaire d'une autre table.

-- Tables liées
CREATE TABLE classes (
  id INT PRIMARY KEY,
  nom VARCHAR(20)
);

CREATE TABLE eleves (
  id INT PRIMARY KEY,
  nom VARCHAR(50),
  id_classe INT,
  FOREIGN KEY (id_classe) REFERENCES classes(id)
);

-- ❌ Erreur: id_classe n'existe pas
INSERT INTO eleves VALUES (1, 'Ali', 999); -- 999 n'existe pas!

ON DELETE CASCADE / ON UPDATE CASCADE — Actions en Cascade

Supprime ou met à jour automatiquement les lignes filles lors de changements dans la table mère.

-- Créer la relation avec cascade
ALTER TABLE eleves 
ADD CONSTRAINT fk_classe 
FOREIGN KEY (id_classe) REFERENCES classes(id) 
ON DELETE CASCADE 
ON UPDATE CASCADE;

-- Exemple avec CASCADE:
DELETE FROM classes WHERE id = 1;
-- ✓ Tous les élèves de cette classe sont automatiquement supprimés!

-- Sans CASCADE (erreur):
-- ❌ Cannot delete class because students are linked

💡 Exemple Complet: Base Structurée

Illustration complète avec contraintes imbriquées:

-- Créer la base
CREATE DATABASE school;
USE school;

-- Table parent (classes)
CREATE TABLE classes (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nom VARCHAR(20) NOT NULL UNIQUE,
  salle INT CHECK (salle > 0),
  date_creation DATE DEFAULT CURDATE()
);

-- Table enfant (élèves)
CREATE TABLE eleves (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nom VARCHAR(50) NOT NULL,
  email VARCHAR(100) UNIQUE,
  note DECIMAL(4,2) CHECK (note >= 0 AND note <= 20),
  id_classe INT NOT NULL,
  date_inscription DATETIME DEFAULT NOW(),
  actif BOOLEAN DEFAULT TRUE,
  FOREIGN KEY (id_classe) REFERENCES classes(id) ON DELETE CASCADE
);

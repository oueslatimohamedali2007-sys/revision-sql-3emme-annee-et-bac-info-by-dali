Chapitre 2

Types de données

Les principaux formats de données utilisables dans les colonnes pour structurer vos informations.

Types Numériques — INT

Stocke des nombres entiers (positifs ou négatifs).

-- Entier classique (-2^31 à 2^31-1)
age INT,
id_eleve INT,

-- Variations
TINYINT,   -- Plus petit: -128 à 127
SMALLINT,  -- Petit: -32768 à 32767
BIGINT,    -- Grand: -2^63 à 2^63-1

-- Entier non-signé (0 et positif)
nombre_eleves INT UNSIGNED,

Types Numériques — DECIMAL et FLOAT

Stocke les nombres décimaux avec précision.

-- Décimal précis (recommandé pour l'argent)
prix DECIMAL(10, 2),        -- 10 chiffres total, 2 après la virgule
note DECIMAL(4, 2),         -- Ex: 15.50
-- Plage: -99999999.99 à 99999999.99

-- Décimal alternatif
montant NUMERIC(8, 3),      -- Équivalent à DECIMAL

-- Nombre flottant (moins précis mais plus rapide)
pourcentage FLOAT,
temperature DOUBLE,

Types de Chaînes — VARCHAR et CHAR

Stocke du texte avec longueur variable ou fixe.

-- Chaîne de longueur variable (recommandée)
nom VARCHAR(50),            -- Jusqu'à 50 caractères
email VARCHAR(100),
description VARCHAR(255),

-- Chaîne de longueur fixe
code_postal CHAR(5),        -- Exactement 5 caractères
code_pays CHAR(2),          -- Codes pays: FR, ES, IT...

-- Très long texte
contenu TEXT,               -- Jusqu'à 65,535 caractères
bio LONGTEXT,              -- Jusqu'à 4 Go

Types de Dates et Heures

Formats pour enregistrer le temps avec différentes précisions.

-- Date seule (format: YYYY-MM-DD)
date_naissance DATE,        -- Ex: 2008-05-15
date_inscription DATE,

-- Heure seule (format: HH:MM:SS)
heure_arrivee TIME,         -- Ex: 14:30:45

-- Date ET Heure ensemble
date_creation DATETIME,     -- Ex: 2026-05-19 14:30:45
dernier_acces TIMESTAMP,    -- Avec fuseau horaire

-- Temps écoulé (intervalle)
duree_cours INTERVAL,       -- Ex: 2 heures 30 minutes

Types Spécialisés

Types pour des besoins particuliers:

-- Booléen (0 ou 1)
actif BOOLEAN,              -- TRUE ou FALSE
est_valide BOOL,

-- Énumération (liste de valeurs fixes)
genre ENUM('M', 'F', 'Autre'),
statut ENUM('Inscrit', 'Actif', 'Absent', 'Diplômé'),

-- Ensemble de valeurs
competences SET('SQL', 'Python', 'JavaScript', 'C++'),

-- Données binaires
image BLOB,                 -- Fichier binaire
signature MEDIUMBLOB,

💡 Table Complète d'Exemple

Exemple réel d'une table avec différents types:

CREATE TABLE eleves (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nom VARCHAR(50) NOT NULL,
  prenom VARCHAR(50),
  date_naissance DATE,
  classe VARCHAR(10),
  note DECIMAL(4, 2),
  email VARCHAR(100) UNIQUE,
  telephone CHAR(12),
  statut ENUM('Inscrit', 'Actif', 'Absent'),
  actif BOOLEAN DEFAULT TRUE,
  date_inscription DATETIME DEFAULT NOW(),
  comments TEXT,
  INDEX idx_nom (nom),
  INDEX idx_classe (classe)
);
